## Why This Matters

**Previous positioning:** MCLI was presented primarily as a workflow framework where you register scripts.

**New positioning:** MCLI is a universal script runner that can also manage workflows.

**User journey:**
1. **Start simple:** Run any script with `mcli run ./script.py`
2. **Add convenience:** Use tab completion to discover scripts
3. **Go advanced:** Optionally register scripts as workflows for scheduling/sharing

This makes MCLI more accessible to new users while maintaining all advanced features for power users.
